A different shape

You've seen the layout before: a list of sessions down the left, an agent chat in the middle, the diff on the right. Almost every tool ships the same three panes.

Architect takes a different shape. Picture the Architect's room in The Matrix Reloaded — one chair, a wall of monitors, every screen showing something else. That's the model: a grid of real terminals, each running its own agent, each able to call for you on its own. Glance at the wall, find the screen that's lit, go there.

And it's all built in. The agent-awareness, the diff view, directory switching, spawning terminals — no plugins to install, no obscure config files to wire together. It works the moment you open it.

What it does

Status highlights

Every terminal is agent-aware. A cell shifts its tint and animates the instant its agent finishes or stops for approval, so one look at the grid tells you where to go. Detection runs on agent hooks, so it isn't tied to a single CLI.

A grid of terminals

Start with one terminal; ⌘N spawns another and the grid reflows, ⌘W closes one and it compacts forward. ⌘Enter blows a cell up to full screen, ⌘Arrow and ⌘1–⌘0 move focus. Spawn, switch, and close without leaving the keyboard.

MCP support

Architect runs its own MCP server, so MCP-aware apps can spawn a terminal in any directory on their own. Hand a task off to a worktree and a fresh agent cell shows up — no manual setup.

Diff in a keystroke

⌘D pulls up a repo-wide git diff across staged, unstaged, and untracked changes. Click any line to leave an inline comment, then send the whole batch to the agent to act on.

Recent folders

⌘O jumps back into a directory you've worked in. Type to filter the list, with substring highlighting, arrow-key navigation, and ⌘1–⌘9 to pick one instantly.

Worktree picker

⌘T to cd into a git worktree for parallel work on a separate branch, or create a new one on the spot. It works by sending plain git commands to the terminal — fully traceable, no magic.
